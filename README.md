# DesafioBackend_Criptografia.
Este projeto foi feito para estudo e aperfeiçoamento das ferramentas Java, Spring e Criptografia de dados sensíveis. A ideia partiu do Desafio de Criptografia do repositório do Backend Brasil.

## Sobre o projeto.
Neste projeto foi criado uma aplicação de CRUD simples em Java/Spring seguindo o Padrão MVC (Model View Controller) com o desafio de criptografia de dados sensíveis, também foi utilizado o banco de dados PostgreSQL para persistência de dados. 

## Sobre o desafio.
No desafio se pede que dado uma entidade que contém em sua estrutura dados sensíveis utilize em sua manipulação uma criptografia dos dados. É utilizado na aplicação o algoritmo SHA-256 contido na biblioteca java.security. No escopo da entidade os campos sensíveis devem ser o "userDocument" e "creditCardToken". 

Mais detalhes do desafio você pode encontrar no reposítorio do [Backend Brasil](https://github.com/backend-br/desafios).
